## 功能1：专利新颖性检查（Patent Novelty Check）

### 问题
当前Pipeline生成的分子可能是已知化合物（已有专利或文献报道）。如果投入合成后发现是已知的，时间和资金都浪费。

### 解决方案
对每个Top候选分子，查询PubChem数据库：
- 如果SMILES在PubChem中存在 → 标记为"已知化合物"，提供CID和文献链接
- 如果不存在 → 标记为"新颖化合物"，高优先级推进

### 实现
- 新增工具：`check_patent_novelty(smiles_list)`
- 调用PubChem PUG-REST API：`https://pubchem.ncbi.nlm.nih.gov/rest/pug/compound/smiles/{smiles}/cids/JSON`
- 返回：每个分子的新颖性状态、PubChem CID（如果存在）、文献计数
- 在`get_top_molecules`后自动调用，或作为独立步骤

### 用户交互

Agent: "Top分子已生成，正在检查专利新颖性..."
Agent: "3个分子中，2个为新颖化合物（高优先级），1个与已知化合物相似（需谨慎）"
```

## 功能2：多靶点选择性分析（Selectivity Analysis）

### 问题
激酶抑制剂常缺乏选择性，抑制多个同家族靶点导致毒性。例如EGFR抑制剂如果同时抑制野生型EGFR，毒性就很大。

### 解决方案
对候选分子，除了对接主靶点，还对同家族脱靶做对接：
- 已知靶点家族映射（如EGFR家族：EGFR, ERBB2, ERBB4, ERBB3）
- 对每个脱靶运行简化对接
- 计算选择性指数：S = IC50(脱靶) / IC50(主靶) > 100 为好

### 实现
- 新增工具：`analyze_selectivity(project_id, smiles, target_family)`
- 靶点家族映射表（从target_database扩展）
- 对每个家族成员做对接，计算ΔΔG
- 返回：选择性矩阵、风险评估

### 用户交互
```
用户: "这个分子选择性怎么样？"

Agent: "- EGFR WT: 对接分数-6.5 (选择性 12x，风险：中)"
Agent: "- ERBB2: 对接分数-5.8 (选择性 28x，风险：低)"

## 功能3：ADMET结构-活性关系分析（SAR for ADMET）

### 问题
当前ADMET只给一个分数，但不知道**为什么**ADMET差。用户需要知道是哪个基团导致毒性/溶解度差。

### 解决方案
分析Top分子的结构特征与ADMET属性的关系：
- 找出与低ADMET分数相关的结构基团（如硝基→毒性、长链→渗透性差）
- 基于QSAR模型给出基团替换建议
- 参考已知的ADMET结构规则（如hERG规则：含氟芳香环→风险）

### 实现
- 新增工具：`analyze_admet_sar(project_id)`
- 输入：Top分子的SMILES + ADMET分数
- 分析：结构片段与ADMET属性的相关性（用RDKit提取子结构）
- 输出：问题基团列表 + 建议替换基团

### 用户交互
```

Agent: "分析Top 5分子的ADMET结构-活性关系："

Agent: "2. 分子#1 LogP=4.8，超过推荐范围（渗透性差）"
Agent: "3. 建议：将硝基替换为酰胺，预计AMES风险降低60%"

## 功能4：合成路线可行性评估（Synthetic Route Assessment）

### 问题
SA Score只是复杂度评分，不知道**具体怎么合成**、需要多少步骤、起始物料是否可获得。有些分子看起来不错但合成路线极长。

### 解决方案
基于逆合成算法（如Retro*或简化规则）生成合成路线：
- 从目标分子出发，逐步拆解为可购买的起始物料
- 评估：步骤数、反应类型、起始物料可获得性、估计成本
- 标记高风险步骤（如需要剧毒试剂、无水条件等）

### 实现
- 新增工具：`assess_synthesis_route(smiles)`
- 使用简化逆合成规则库（如RDKit的 retrosynthetic 分析）
- 或调用外部API（如IBM RXN、PostEra）
- 返回：推荐路线、步骤数、风险等级、成本估算

### 用户交互
```
用户: "这个分子能合成吗？"
Agent: "合成路线评估："
Agent: "- 推荐路线：5步合成，起始物料可购买"
Agent: "- 步骤1：Suzuki偶联（低风险）"
Agent: "- 步骤3：需要无水条件（中等风险）"
Agent: "- 估计成本：$500/g（实验室规模）"

## 功能5：分子聚类与多样性（Molecular Clustering & Diversity）

### 问题
当前Top分子可能只是同一骨架的微小变化。如果该骨架的假设错误（如结合模式不对），所有候选分子都失败，全军覆没。

### 解决方案
对Top分子进行聚类分析：
- 基于Tanimoto相似度或Murcko骨架聚类
- 确保Top分子覆盖不同骨架类别
- 对每个聚类中心给出代表性分子
- 如果只有一个聚类，建议增加多样性（调整生成参数）

### 实现
- 新增工具：`analyze_molecular_diversity(project_id, smiles_list)`
- 使用RDKit计算指纹相似度矩阵
- 聚类（层次聚类或K-means）
- 输出：聚类数、每类代表分子、骨架类型、多样性评分

### 用户交互
```
用户: "这些分子结构多样吗？"
Agent: "Top 10分子聚类分析："
Agent: "- 聚类1（4个分子）：苯并咪唑骨架，代表性：分子#3"
Agent: "- 聚类2（3个分子）：喹唑啉骨架，代表性：分子#7"
Agent: "- 聚类3（2个分子）：吡啶并嘧啶骨架，代表性：分子#9"
Agent: "- 孤立分子（1个）：独特骨架，建议优先验证"
Agent: "多样性评分：0.72（良好，覆盖3个主要骨架）"
```

## 实现优先级

```
功能3（ADMET-SAR）→ 功能5（多样性）→ 功能1（专利查重）→ 功能2（选择性）→ 功能4（合成评估）
```

理由：
- 功能3、5：纯RDKit分析，不需要外部API，实现最快，价值明确
- 功能1：需要PubChem API，有网络依赖，但价值高
- 功能2：需要扩展靶点数据库和多次对接，计算量大
- 功能4：需要逆合成算法，最复杂，但价值最高（避免合成失败）
