# Agent 输出排版规范

> 本文档规定 DrugDesign Copilot Agent 所有输出内容的排版标准，确保输出美观、专业、易读。

---

## 一、核心原则

1. **数据优先**：用户要看的是数据，不是文字描述。表格 > 列表 > 段落。
2. **层次分明**：用加粗和表格区分层次，不用 `##` / `###` 标题。
3. **减少 emoji**：专业报告中用 "通过/需关注" 替代 "✅/⚠️"，用 "高/低" 替代 "🔴/🟢"。
4. **一页看完**：关键信息集中在首屏，不需要滚动才能看到核心结论。
5. **评价具体**：每个数据旁边要有评价（如 "高风险/低风险"），不要只给数字。

---

## 二、禁止使用的元素

| 禁止元素 | 原因 | 替代方案 |
|----------|------|----------|
| `##` / `###` / `####` | 层级太深，视觉混乱 | `**加粗**` 做标题 |
| `---` / `***` 分隔线 | 占用空间，无信息价值 | 空行分隔 |
| 大量 emoji (✅⚠️🔴🟢) | 不专业，不同系统显示不一致 | 文字评价 (通过/需关注/高/低) |
| `-` / `*` 列表超过 5 项 | 难以扫描，信息密度低 | Markdown 表格 |
| `>` 引用块 | 不适用于数据展示 | 直接展示 |

---

## 三、推荐排版格式

### 3.1 标题层级

```markdown
**一级标题** — 大标题，报告名称

**二级内容** — 分组标题，如"吸收""毒性"

正文内容...
```

**规则**：
- 只用 `**加粗**` 做标题，不用 `#` 系列
- 一级和二级之间用空行分隔

### 3.2 数据展示

**推荐：三列表格**（指标 + 数值 + 评价）

```markdown
| 指标 | 数值 | 评价 |
|------|------|------|
| 分子量 (MW) | 375.2 | 通过 |
| LogP | 3.8 | 通过 |
| hERG | 0.4 | 低风险 |
```

**规则**：
- 每个表格必须有表头（指标/数值/评价）
- 数值列统一精度（小数位一致）
- 评价列必须给出判断，不能只给数字

### 3.3 评价用词规范

| 场景 | 推荐用词 | 避免用词 |
|------|----------|----------|
| 通过标准 | 通过 / 需关注 / 超标 | ✅ / ⚠️ / 失败 |
| 风险等级 | 高风险 / 低风险 / 阴性 / 阳性 | 🔴 / 🟢 / 正常 |
| 质量等级 | 优秀 / 良好 / 一般 / 需改进 | 完美 / 差 / 不行 |
| 趋势判断 | 偏高 / 偏低 / 适中 / 良好 | 高 / 低 / 正常 |

```markdown

| 分子量 (MW) | 375.2 | 通过 |
| LogP | 3.8 | 通过 |
| QED | 0.747 | 良好 |

**吸收 (Absorption)**
| 指标 | 数值 | 评价 |
|------|------|------|
| 水溶性 | -2.52 | 偏低 |
| 渗透性 | -4.19 | 偏低 |
| 口服生物利用度 | 100% | 高 |

**毒性 (Toxicity)**
| 指标 | 数值 | 评价 |
|------|------|------|
| hERG | 40% | 低风险 |
| AMES | 30% | 阴性 |

```

```markdown
**失败分子分析报告**

共发现 15 个失败分子

**失败阶段分布**
| 失败阶段 | 数量 | 占比 |
```

## 五、排版检查清单

每次修改格式化代码后，检查：

- [ ] 没有使用 `##` / `###` / `---` / `***`
- [ ] 没有使用 emoji（✅⚠️🔴🟢）
- [ ] 数据用表格展示，不是列表
- [ ] 每个数值旁边有评价（通过/需关注/高/低）
- [ ] 标题用 `**加粗**`，不是 `#` 系列
- [ ] 总体评价在首屏可见，不需要滚动
- [ ] 数值精度统一（如都保留 2 位小数）
- [ ] 表格有对齐的列宽（指标/数值/评价）

## 六、常见错误示例

### 错误示例 1：使用 ## 标题

```markdown
## 基础理化性质
- 分子量: 375.2
- LogP: 3.8
```

**问题**：`##` 层级太深，`-` 列表难以扫描。

**正确**：
```markdown
**基础理化性质**
| 指标 | 数值 | 备注 |
|------|------|------|
| 分子量 (MW) | 375.2 | 通过 |
| LogP | 3.8 | 通过 |
```

### 错误示例 2：使用 emoji

```markdown
| 规则 | 结果 |
|------|------|
| Lipinski | 通过 ✅ |
| PAINS | 无 ⚠️ |
```

**问题**：emoji 不专业，在某些系统显示为方块。

**正确**：
```markdown
| 规则 | 结果 | 状态 |
|------|------|------|
| Lipinski | 通过 | 通过 |
| PAINS | 无 | 通过 |
```

### 错误示例 3：只有数字没有评价

```markdown
| 指标 | 数值 |
|------|------|
| hERG | 0.4 |
| AMES | 0.3 |
```

**问题**：用户不知道 0.4 是高还是低。

**正确**：
```markdown
| 指标 | 数值 | 评价 |
|------|------|------|
| hERG | 40% | 低风险 |
| AMES | 30% | 阴性 |
```

---

## 七、总结

> **核心口诀**：表格代替列表，加粗代替标题，文字代替 emoji，评价伴随数据。

所有 Agent 输出格式化代码（`executor.py` 中的 `_format_*` 方法）必须遵循本文档规范。
